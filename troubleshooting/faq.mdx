<AccordionGroup>
  <Accordion title="비밀번호를 잊어버렸어요">
    로그인 화면의 **비밀번호 찾기**를 이용하거나, 조직 관리자에게 비밀번호 초기화를 요청하세요.  
    SSO를 사용하는 경우 IdP(예: Azure AD)에서 비밀번호를 재설정해야 합니다.
  </Accordion>

  <Accordion title="초대 메일이 오지 않아요">
    - 스팸 메일함을 확인하세요.  
    - 이메일 주소가 정확한지 관리자에게 확인하세요.  
    - 회사 메일 필터 정책으로 차단될 수 있으니 IT 담당자에게 `noreply@audigate.com` 도메인 허용을 요청하세요.
  </Accordion>

  <Accordion title="파일 업로드가 실패해요">
    - 파일 크기 제한(기본 50MB)을 확인하세요.  
    - 지원 형식(PDF, XLSX, DOCX, PNG, JPG 등)인지 확인하세요.  
    - 네트워크 연결 상태를 확인하세요.
  </Accordion>

  <Accordion title="권한이 없다는 메시지가 나와요">
    해당 기능에 대한 역할(Role) 권한이 없을 수 있습니다. 관리자에게 필요한 권한 부여를 요청하세요.
  </Accordion>

  <Accordion title="AI 질의에서 결과가 없어요">
    - LiveMenu에서 **올바른 DB 서비스**가 선택되었는지 확인하세요.
    - 질문의 **데이터 분류**(개인정보, 고객정보 등)에 해당 테이블이 **감사 정책 작성** 시 등록되었는지 확인하세요. 분류 미등록 오브젝트는 분류 질의에 포함되지 않습니다.
    - 조회 **기간**이 너무 짧거나, 해당 기간에 감사 이벤트가 수집되지 않았을 수 있습니다.
    - **해석된 조건 (parsedIntent)** 을 펼쳐 서버가 이해한 기간·분류가 의도와 맞는지 확인하세요.
  </Accordion>

  <Accordion title="AI 질의 LLM 연결 실패가 표시돼요">
  - 서버 `audigate.conf`의 `nlLlm.endpoint`·`model`이 Ollama(또는 호환 API)와 일치하는지 확인하세요.
  - Ollama 프로세스가 기동 중인지, 방화벽이 로컬 포트를 막지 않는지 확인하세요.
  - LLM 없이도 **규칙 파서**로 단순 패턴 질문은 동작합니다. `nlLlm.enabled: false`로 규칙만 사용할 수도 있습니다.
  </Accordion>

  <Accordion title="AI 질의에서 사용자 ID가 암호문으로 보여요">
    콘솔 사용자에게 **auditIdDecript** 기능 권한이 없으면 사용자 식별자가 마스킹됩니다. 순위·건수는 정확하지만 평문 ID가 필요하면 관리자에게 **authPrivs → auditIdDecript** 권한 부여를 요청하세요.
  </Accordion>

  <Accordion title="감사 정책 적용 시 데이터 분류를 요구해요">
    **스키마·오브젝트(OBJ) 감사**는 AI 질의·대시보드 연동을 위해 **데이터 분류 1개 이상** 선택이 필수입니다. **데이터 분류 사전 등록** 모달에서 분류를 고른 뒤 정책을 적용하세요.
  </Accordion>

  <Accordion title="모바일에서도 사용할 수 있나요?">
    Audigate는 반응형 웹으로 모바일 브라우저에서 조회가 가능합니다.  
    다만 복잡한 편집 작업은 데스크톱 사용을 권장합니다.
  </Accordion>
</AccordionGroup>
