Audigate Server 웹 콘솔에 로그인한 뒤, 상단 메뉴에서 각 기능 화면을 열어 DB 감사·게이트·시스템을 운영합니다. 본 문서는 **현재 등록된 메뉴 구조**를 기준으로 화면 구성과 주요 버튼·그리드를 설명합니다.

<Info>
  메뉴 노출 여부는 콘솔 사용자 권한(`authMenus`)에 따라 달라집니다. 아래에 없는 메뉴는 관리자에게 접근 권한을 요청하세요.
</Info>

## 로그인

1. 조직에서 안내받은 **Audigate Server URL**에 접속합니다.
2. **사용자 ID**와 **비밀번호**를 입력하고 로그인합니다.
3. 인증에 성공하면 **메인 콘솔**(`main.jsp`)이 열리고, 상단에 **감사 · 게이트 · 관리자** 메뉴가 표시됩니다.

<Frame>
  ![Image](/images/image-22.png)
</Frame>

| 항목 | 설명 |
| --- | --- |
| **최초 설치** | 서버가 아직 설치되지 않은 경우 설치 안내 페이지로 이동할 수 있습니다. |
| **비밀번호** | 전송 전 암호화 처리됩니다. 분실 시 관리자에게 초기화를 요청하세요. |
| **로그인 실패** | ID·비밀번호 오류 또는 계정 잠금(`LOCKED`) 상태를 확인하세요. |

## 콘솔 공통 UI

로그인 후 모든 화면에서 공통으로 사용하는 요소입니다.

| 영역 | 설명 |
| --- | --- |
| **상단 메뉴** | **감사**, **게이트**, **관리자** 드롭다운과 우측 **나의 계정**, **패스워드**, **전체화면**, **로그아웃** |
| **LiveMenu** | DB 서비스(인스턴스)를 선택하는 좌측 서비스 메뉴. 감사·게이트 대부분 화면에서 사용 |
| **툴바** | 화면마다 **추가**, **편집**, **삭제**, **새로고침**, **검색**, **저장** 등 |
| **그리드** | 목록 조회·정렬·필터. 행 클릭 시 우측 또는 하단 **상세 패널**에 내용 표시 |

<Tip>
  서비스를 먼저 선택해야 하는 화면에서는 LiveMenu에서 대상 DB를 고른 뒤 작업하세요. 서비스 목록은 **관리자 → 디비 서비스 관리자**에서 등록합니다.
</Tip>

## 감사

상단 **감사** 메뉴에 등록된 화면입니다. Oracle 감사 정책·로그 수집·분석을 담당합니다.

### 통합 거버넌스 대시보드

**목적:** 전체 DB 서비스를 집계한 Executive KPI·Health·이상행위 요약을 한 화면에서 확인합니다.

| UI 요소 | 설명 |
| --- | --- |
| **새로고침** | 위젯 데이터를 수동으로 다시 불러옵니다. |
| **위젯 카드** | 실사용자 귀속률, Gate 활성, 이상 알람, 민감 접근, 미귀속 접근, 수집 Lag, Collector 상태, ES 상태, WS 연결 등 |
| **? (도움말)** | 각 카드 우측의 도움말 아이콘으로 지표 정의를 확인합니다. |
| **자동 갱신** | 약 60초 간격으로 위젯이 자동 갱신됩니다. |

**사용 흐름:** 메뉴 진입 → 전체 서비스 기준 위젯 자동 로드 → 이상 지표 확인 → 필요 시 **감사 로그 분석** 등 상세 화면으로 이동

| 위젯 예시 | 설명 |

| **Collector / 수집 Lag** | 수집기 상태 및 지연 시간 |

### 감사 정책 작성

**목적:** Oracle `AUDIT` / `NOAUDIT` 정책 SQL을 화면에서 구성하고 대상 DB에 적용합니다.

| UI 영역 | 설명 |
| --- | --- |
| **LiveMenu** | 정책을 적용할 DB 서비스 선택 |
| **초기화** | 입력한 정책 조건을 초기 상태로 되돌립니다. |
| **정책 적용** | 작성된 SQL 미리보기 확인 후 DB에 적용합니다. |
| **감사 유형** | **문장/권한 감사**(STMT) 또는 **객체 감사**(OBJ) 선택 |
| **문장 감사 옵션** | 감사 대상 문장 유형, DB 사용자(By) 다중 선택 |
| **객체 감사 옵션** | SQL 연산(SELECT, INSERT 등), 스키마·오브젝트, DIRECTORY 등 대상 지정 |
| **Session / 성공 조건** | Session·Access, WHENEVER SUCCESSFUL / NOT SUCCESSFUL |
| **SQL 미리보기** | 적용될 `AUDIT` 문장을 우측에서 확인 |

**사용 흐름:** 서비스 선택 → 감사 유형·대상·조건 설정 → SQL 미리보기 확인 → **정책 적용** → 확인 모달에서 요약 검토 후 확정

<Warning>
  정책 적용은 운영 DB에 직접 반영됩니다. 적용 전 SQL과 대상 오브젝트를 반드시 확인하세요.
</Warning>

### 감사 정책 현황

**목적:** 과거 정책 적용 **이력** 조회와 Oracle **현재 감사 옵션** 확인·`NOAUDIT` 적용을 수행합니다.

| UI 영역 | 설명 |
| --- | --- |
| **기간** | 조회 시작·종료 일시 (기본 약 7일) |
| **검색** | 선택 기간·유형에 맞는 이력을 조회합니다. |
| **감사유형 탭** | 전체 / STMT / PRIV / OBJ 전환 |
| **적용 이력 그리드** | appliedAt, consoleUser, policySummary, auditSql, success (50건/페이지) |
| **현재 상태 그리드** | Oracle `DBA_*_AUDIT_OPTS` 뷰 기반 현재 옵션. OBJ 탭은 ALT, SEL, INS 등 동작별 컬럼 표시 |

**그리드 조작**

- 이력 행 클릭 시 해당 감사 유형 탭으로 자동 전환됩니다.
- **현재 상태** 그리드 **우클릭** → `NOAUDIT` 메뉴 (OBJ는 동작별 또는 ALL)

**사용 흐름:** 서비스·기간 선택 → **검색** → 이력 확인 → 탭 전환으로 현재 옵션 조회 → 필요 시 우클릭으로 NOAUDIT 적용

### 감사 로그 수집·보관 관리

**목적:** DB 감사 로그의 **수집 주기**, **보관**, **Purge**, **테이블스페이스**를 관리합니다.

| 카드 | 설정 항목 |
| --- | --- |
| **테이블스페이스** | Trail, Trail 크기, 현재 TS, 새 TS(드롭다운). 기본 TS 사용 시 경고 표시 |
| **수집** | 수집 간격: 10초 / 30초 / 1분 / 5분 / 10분 |
| **보관** | 기본 정리 주기(시간, 1~8760) |
| **Purge** | Purge 주체(AUDIGATE / DBMS / BOTH), DB Purge 간격, 배치 크기(100~1,000,000), Job 상태 |

각 카드마다 **새로고침**, **저장** 버튼이 있습니다. 테이블스페이스는 변경된 항목만 저장됩니다.

**사용 흐름:** 서비스 선택 → 카드별 현재 설정 확인 → 값 변경 → **저장**

### 감사 로그 분석

**목적:** Elasticsearch에 수집된 감사 로그를 검색·분석합니다.

| UI 영역 | 설명 |
| --- | --- |
| **LiveMenu** | 조회 대상 DB 서비스 |
| **기간** | 기본 최근 24시간 (`YYYY-MM-DD HH:mm`) |
| **More** | 고급 검색 조건 패널 열기 |
| **검색** | 조건에 맞는 로그 조회 (100건/페이지) |
| **메인 그리드** | 감사 로그 컬럼 표시 (아래 주요 컬럼 참고) |
| **상세 패널** | 행 선택 시 우측에 전체 필드 상세 표시 |
| **컬럼 필터** | 각 컬럼 헤더 아래 텍스트 필터로 클라이언트 추가 필터링 |

**주요 그리드 컬럼**

| 컬럼 | 설명 |
| --- | --- |
| **Time / TIMESTAMP#** | 이벤트 발생 시각 |
| **USERID / USERHOST / TERMINAL** | DB·OS 접속 식별 정보 |
| **ACTION / ACTION NAME** | 감사 Action 코드·이름 |
| **STATEMENT / SQLTEXT** | 실행 문장·SQL 본문 |
| **OBJ\$OWNER** / **OBJ\$NAME** | 대상 오브젝트 |
| **CLIENTID / WSESSIONID** | WAS 세션·실사용자 귀속 키 |
| **RETURNCODE** | SQL 실행 결과 코드 |

**고급 조건 (More):** 필드 \+ 연산자(`=`, `IN`, `LIKE`) \+ 값을 행 단위로 추가·초기화할 수 있습니다.

**사용 흐름:** 서비스·기간 선택 → (선택) More 조건 추가 → **검색** → 행 클릭으로 상세 확인 → 컬럼 필터로 추가 좁히기

## 게이트

상단 **게이트** 메뉴. DB 접근 승인·AP-Role(애플리케이션 롤) lifecycle을 관리합니다.

### DB 사용자 및 결재선 관리자

**목적:** DB 접근 사용자 목록과 **결재선(approverId)** 트리를 구성합니다.

| UI 영역 | 설명 |
| --- | --- |
| **편집** | 선택한 사용자 1명의 상세 정보를 모달에서 수정합니다. |
| **새로고침** | 사용자 목록을 다시 불러옵니다. |
| **결재선 저장** | 드래그로 변경한 결재선 구조를 저장합니다. |
| **검색** | memberId, 이름 등으로 목록 필터링 |

**그리드 컬럼:** memberId(트리), memberName, deptName, osUsername, ipAddress, terminal, email, mobileNumber, lang, accountStatus, isLogin, approverId, created, updated, expiryDate, lastLogin

**트리 조작**

- **드래그 앤 드롭**으로 결재자(approverId) 변경. 루트 드롭존에 놓으면 최상위 결재자가 됩니다.
- Ctrl / Shift로 동일 레벨 다중 선택 가능
- **더블클릭**으로 편집 모달 열기

**편집 모달 필드:** 이름, 부서, 이메일, 휴대폰, 언어(ko/en), approverId, 계정상태(LOCKED/OPEN)

**사용 흐름:** 목록 로드 → 사용자 선택 → 드래그로 결재선 편집 → **결재선 저장** 또는 개별 **편집** 후 저장

### 게이트 롤 매니저

**목적:** 승인·요청된 **AP-Role**(애플리케이션 롤)을 등록·변경·삭제합니다.

| 버튼 | 설명 |
| --- | --- |
| **추가** | 새 AP-Role 생성 |
| **편집** | 선택 롤 수정 |
| **삭제** | 선택 롤 삭제 |
| **복제** | 기존 롤을 복사하여 새 롤 생성 |
| **새로고침** | 목록 갱신 |
| **검색** | 목록 필터 |

**목록 그리드 컬럼:** roleId, memberId, memberName, serviceName, status, enabled, expiryDate, isValid

**롤 상세·폼 주요 항목**

- 사용자 선택(피커), 대상 DB 서비스
- **권한 설정(4단계):** DB 서비스 → 스키마 → 오브젝트 → 권한
- requestSql, enabled, expiryDate
- 검증 항목: OS User, IP, Terminal
- grantDcls, authCode, managerIds, comments 등

**권한 유형 예시:** TABLE/VIEW — SELECT·UPDATE·INSERT·DELETE / SEQUENCE — SELECT / PROCEDURE·PACKAGE·FUNCTION — EXECUTE·DEBUG

**사용 흐름:** **추가** → 사용자·권한(Grant) 설정 → 저장 / 기존 롤 **편집**·**복제**·**삭제**

## 관리자

상단 **관리자** 메뉴. 시스템·콘솔 계정·메타데이터를 설정합니다. 일반 사용자에게는 권한이 없을 수 있습니다.

### 디비 서비스 관리자

**목적:** Audigate가 관리할 **DB 서비스(인스턴스)** 를 등록·변경·삭제합니다.

| 버튼 | 설명 |
| --- | --- |
| **추가 / 편집 / 삭제 / 복제** | 서비스 CRUD |
| **새로고침 / 검색** | 목록 갱신·필터 |
| **연결 테스트** | 저장 전 DB 연결 검증 (성공 시에만 저장 가능) |

**그리드 컬럼:** serviceName, group, dbEngine, connectUrl, hostName, username, version, isActive, color, registDate

**폼 주요 필드:** serviceName(영숫자), dbEngine(ORACLE/MYSQL/HANA/POSTGRESQL), connectUrl, username/password, color, description

**Oracle 전용:** 편집 시 NAME/VALUE 파라미터 그리드, `schemas_using_gate` 스키마 다중 선택

**사용 흐름:** **추가** → 연결 정보 입력 → **연결 테스트** 성공 → **저장**

### 그룹 관리자

**목적:** DB 서비스를 **그룹**으로 묶어 LiveMenu 등에 표시합니다.

| UI 영역 | 설명 |
| --- | --- |
| **좌측** | 그룹 목록 (ETC 제외) \+ **새 그룹 추가** |

**목적:** **웹 콘솔** 로그인 계정과 메뉴·서비스·기능 권한을 관리합니다.

### 메타 오브젝트 관리자

**목적:** Elasticsearch **인덱스·템플릿**을 관리합니다. 감사 로그 검색 인프라 유지보수용입니다.

| 탭 | 설명 |
| --- | --- |
| **템플릿** | index template 목록·매핑·설정 |
| **인덱스** | 실제 인덱스 health, docCount, size 등 |

**템플릿 그리드:** templateName, description, mapping, upgradeMapping, indexCount, settings

**인덱스 그리드:** indexName, description, health(green/yellow/red), created, mapping, upgrade, docCount, size, settings

**행 액션 (확인 후 실행):** recreate, truncate(인덱스만), fixReplicas(health=yellow), drop

<Warning>
  drop·truncate·recreate는 데이터 손실이 발생할 수 있습니다. 운영 환경에서는 반드시 백업·영향 범위를 확인하세요.
</Warning>

## 권장 업무 흐름

<Steps>
  <Step title="업무 확인">
    **업무** 메뉴에서 본인에게 할당된 항목을 확인합니다.
  </Step>
  <Step title="자료 검토">
    첨부 파일, 증빙 자료를 열어 검토하고 코멘트를 남깁니다.
  </Step>
  <Step title="상태 변경">
    검토 완료 후 상태를 **승인**, **반려**, **보완 요청** 등으로 변경합니다.
  </Step>
  <Step title="보고서 확인">
    필요 시 **보고서** 메뉴에서 결과를 확인하거나 PDF로 보냅니다.
  </Step>
</Steps>

- **검색**: 상단 검색창에서 업무 ID, 제목, 담당자로 빠르게 찾기
- **필터**: 상태, 기간, 담당 부서별 목록 필터링
- **알림**: 우측 상단 벨 아이콘에서 미처리 알림 확인\

<Steps>
  <Step title="현황 파악">
    **감사 → 통합 거버넌스 대시보드**에서 전체 KPI를 확인하고, 이상 시 **감사 라이브 대시보드**·**감사 로그 분석**으로 상세 조회합니다.
  </Step>
  <Step title="감사 정책 운영">
    **감사 정책 작성**으로 정책을 적용하고, **감사 정책 현황**에서 이력·현재 옵션을 점검합니다.
  </Step>
  <Step title="로그 수집 관리">
    **감사 로그 수집·보관 관리**에서 수집 주기·Purge·테이블스페이스를 설정합니다.
  </Step>
  <Step title="게이트 운영">
    **DB 사용자 및 결재선** → **게이트 롤 매니저**로 접근 권한을 관리하고, **게이트 라이브 대시보드**로 lifecycle을 모니터링합니다.
  </Step>
  <Step title="시스템 설정 (관리자)">
    **디비 서비스 관리자** → **그룹 관리자** → **콘솔 사용자 관리자** 순으로 기반 환경을 구성합니다.
  </Step>
</Steps>
