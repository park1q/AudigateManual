**감사 → 감사 로그 분석** 화면의 **AI 질의** 탭에서는 한국어 자연어 질문으로 감사 로그를 집계·조회할 수 있습니다.  
조건 검색 탭처럼 필드·연산자를 직접 지정하지 않아도, 「1개월간 개인정보를 가장 많이 조회한 사용자는?」과 같은 질문으로 결과를 받을 수 있습니다.

<Info>
  AI 질의는 **Elasticsearch에 수집된 감사 로그**를 대상으로 합니다. LiveMenu에서 **DB 서비스를 먼저 선택**해야 합니다.
</Info>

## 화면 구성

| 영역 | 설명 |
| --- | --- |
| **탭** | **조건 검색** / **AI 질의** — 동일 화면에서 전환 |
| **대상 서비스** | LiveMenu에서 선택한 DB 서비스 이름 표시 |
| **LLM 상태 배지** | 로컬 LLM 연결 여부 (아래 [LLM 상태](#llm-상태) 참고) |
| **샘플 질문** | 접어 두는 예시 질문 버튼 — 클릭 시 입력창에 채워짐 |
| **대화 영역** | 질문·답변(요약, 집계 표, 해석된 조건)이 채팅 형태로 쌓임 |
| **질문 입력** | 자연어 입력. **Enter** 전송, **Shift+Enter** 줄바꿈 |
| **질의 실행** | 질문을 서버로 전송해 분석 시작 |
| **새 대화** | 후속 질의 컨텍스트를 초기화하고 대화를 비움 |

## 사용 흐름

<Steps>
  <Step title="서비스 선택">
    LiveMenu에서 조회할 DB 서비스를 선택합니다.
  </Step>
  <Step title="AI 질의 탭 열기">
    **AI 질의** 탭을 클릭합니다. 상단 배지로 LLM 연결 상태를 확인할 수 있습니다.
  </Step>
  <Step title="질문 입력">
    질문을 입력하거나 **샘플 질문**을 눌러 예시를 불러온 뒤 **질의 실행**을 클릭합니다.
  </Step>
  <Step title="결과 확인">
    요약 문장, 집계 표(순위·건수 등), 필요 시 **해석된 조건 (parsedIntent)** 를 펼쳐 확인합니다.
  </Step>
  <Step title="후속 질문 (선택)">
    「오늘은?」, 「top 3만」, 「급여로 바꿔」처럼 이전 조건을 유지한 채 기간·분류·건수만 바꿀 수 있습니다. 처음부터 다시 하려면 **새 대화**를 누르세요.
  </Step>
</Steps>

## 질문 예시

### 샘플 질문 (화면 내장)

- 1개월간 고객정보 테이블을 가장 많이 조회한 사용자는?
- 1주일간 개인정보 대량 조회 사용자 알려줘
- 최근 7일간 개인정보 감사 이벤트 건수는?

### 지원하는 질의 유형

| 유형 | 질문에 자주 쓰이는 표현 | 결과 |
| --- | --- | --- |
| **접근 상위 사용자** | 가장 많이 조회한 사용자, top N | 기간·분류별 사용자별 조회 횟수 순위 |
| **대량 조회 사용자** | 대량 조회, 과다 조회 | 임계값 이상 조회한 사용자 목록 |
| **이벤트 건수** | 건수, 몇 건 | 기간·분류·액션(SELECT 등)별 총 건수 |
| **이벤트 목록** | 목록, 내역, 리스트 | 조건에 맞는 감사 이벤트 상위 N건 |
| **최근 이벤트** | 가장 최근, 마지막으로 | 해당 분류·액션의 최신 1건 상세 |
| **오브젝트 순위** | 많이 조회된 테이블 | 오브젝트별 접근 횟수 순위 |
| **정의된 테이블** | 정의된 개인정보 테이블, 등록된 테이블 | 감사 정책에 등록된 오브젝트·분류 목록 (`object_registry`) |

### 기간 표현

**상대 기간** (지금 기준)

- 최근 7일, 30일, 1개월, 1주일, 한 달
- 오늘, 어제, 이번 주, 이번 달

**절대 기간** (특정 날짜 구간)

- 6월 12일부터 일주일, 6월 12일 ~ 6월 19일
- 지난주 월요일부터 금요일까지, 이번 주 월요일부터 오늘까지

<Note>
  조회 가능한 최대 기간은 약 **365일**입니다. 미래 날짜는 자동으로 현재 시각까지로 제한됩니다.
</Note>

### 데이터 분류 키워드

질문에 아래와 같은 한글 표현을 넣으면 해당 **데이터 분류**로 필터링합니다.

| 분류 | 질문에 쓸 수 있는 표현 예 |
| --- | --- |
| **개인정보** (PII) | 개인정보, PII |
| **급여** (PAYROLL) | 급여, 연봉 |
| **특허** (PATENT) | 특허 |
| **고객정보** (CUSTOMER) | 고객, 고객정보, 회원, 회원정보 |
| **재무** (FINANCE) | 재무, 회계 |
| **일반** (GENERAL) | (미분류 기본값) |

<Tip>
  「개인정보」 질의에 결과가 없다면, 해당 테이블에 **감사 정책 작성** 시 데이터 분류가 등록되었는지 확인하세요. 분류가 없는 오브젝트는 의도적으로 집계에서 제외됩니다.
</Tip>

## LLM 상태

화면 상단 배지는 서버의 로컬 LLM(Ollama 등) 연결 상태를 나타냅니다.

| 배지 | 의미 |
| --- | --- |
| **로컬 LLM 연결됨** | LLM이 활성화되어 있고 엔드포인트에 연결됨. 모델명이 함께 표시될 수 있음 |
| **규칙 파서 (LLM 비활성)** | LLM 없이 규칙 기반 파서만 사용. 단순·고정 패턴 질문은 동작 |
| **로컬 LLM 연결 실패** | LLM이 설정되어 있으나 연결 불가. 규칙 파서로 fallback 시도 |
| **LLM 상태 확인 중** | 탭 진입 직후 상태 조회 중 |

질의 실행 시 진행 단계(질문 해석 → LLM 조건 추출 → ES 조회)가 표시됩니다. LLM 사용 시 **수십 초** 걸릴 수 있습니다.

## 사용자 식별자 표시

감사 로그의 사용자 식별자(`identifier`, `wsessionid` 등)는 Elasticsearch에 **암호화된 값**으로 저장됩니다. AI 질의 결과 표시 시:

- **auditIdDecript** 기능 권한이 **있으면** — 평문 사용자 ID로 표시
- **없으면** — 암호문 형태로 마스킹 표시 (동일 사용자는 동일 암호문이므로 순위·건수 집계는 정확)

권한이 필요하면 관리자에게 **콘솔 사용자 관리자**의 **authPrivs → auditIdDecript** 부여를 요청하세요.

## 해석된 조건 (parsedIntent)

답변 하단 **해석된 조건**을 펼치면 서버가 이해한 구조화 JSON을 볼 수 있습니다.  
의도(`intent`), 데이터 분류(`categories`), 기간(`periodDays` 또는 `periodFrom`/`periodTo`), `topN`, `bulkThreshold` 등이 포함됩니다. 결과가 기대와 다를 때 검증·재질의에 활용하세요.

## 사전 준비 (관리자)

AI 질의가 분류별로 정확히 동작하려면 다음이 선행되어야 합니다.

1. **감사 정책 작성** — 스키마·오브젝트 감사 적용 시 **데이터 분류** 1개 이상 선택 (→ `object_registry` 등록)
2. **감사 로그 수집** — 해당 정책으로 발생한 이벤트가 Elasticsearch에 적재
3. **(선택) 로컬 LLM** — `audigate.conf`의 `nlLlm` 섹션 설정 및 Ollama 기동

자세한 분류 등록 방법은 [기본 사용법 — 감사 정책 작성](/guides/basic-usage#감사-정책-작성)을, LLM 서버 설정은 [시스템 설정 — AI 질의 LLM](/admin/settings#ai-질의-llm-로컬-llm)을 참고하세요.

## 제한 사항

- 질의는 **검증된 집계 템플릿**만 실행됩니다. LLM이 생성한 임의 SQL·ES 쿼리는 직접 실행되지 않습니다.
- **분류 미등록 오브젝트**는 「개인정보」 등 분류 질의 결과에 포함되지 않습니다.
- 과거에 수집된 로그 중 `object_categories` 태그가 없는 데이터는 일부 질의에서 제외되거나 레거시 패턴으로만 매칭될 수 있습니다.
- 폐쇄망 환경에서는 외부 클라우드 API 없이 **로컬 LLM 또는 규칙 파서**만 사용합니다.
