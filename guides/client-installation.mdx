**Audigate Client**는 DB 직접 접속을 위한 **권한 요청·승인·Role 획득** 데스크톱 애플리케이션입니다.  
승인 완료 후 `apply_my_roles` PL/SQL 스크립트(인증키)를 발급받아 SQL*Plus, SQL Developer 등에서 실행합니다.

<Info>
  Client는 **Audigate Server**의 REST API(`/rest/dbuser`)에 연결합니다. Server 설치·웹 초기 설정이 선행되어야 합니다.
</Info>

## 제공 클라이언트

| 클라이언트 | 기술 스택 | 대상 OS |
| --- | --- | --- |
| **AudigateClient** | React 19 + Tauri 2 | Windows, macOS, Linux |
| **AudigateClient.Net** | WPF + .NET 8 | Windows 10/11 |

운영 배포는 조직 정책에 맞는 클라이언트를 선택합니다. 두 클라이언트 모두 동일 Server REST API를 사용합니다.

## Server URL 설정

Client는 Server REST API 엔드포인트에 연결합니다.

```text
http://<server-host>:<port>/rest/dbuser
```

| 환경 | URL 예시 |
| --- | --- |
| 개발 (Cargo) | `http://localhost:9071/rest/dbuser` |
| 운영 | `https://audigate.example.com/rest/dbuser` |

<Warning>
  URL 끝에 `/rest/dbuser` 경로를 포함해야 합니다. 포트와 프로토콜은 Server 배포 환경에 맞게 설정하세요.
</Warning>

## AudigateClient (Tauri) 설치

### 1. 소스 빌드

```bash
cd AudigateClient
npm install

# 웹 개발 서버 (브라우저 테스트)
npm run dev

# Tauri 데스크톱 개발
npm run tauri:dev

# 프로덕션 설치 패키지 빌드
npm run tauri:build
```

### 2. Server URL 설정

`public/setting.json`을 수정합니다.

```json
{
  "baseServerUrl": "http://192.168.0.25:9071/rest/dbuser",
  "lang": "ko"
}
```

| 속성 | 설명 |
| --- | --- |
| `baseServerUrl` | Server REST API 기본 URL |
| `lang` | UI 언어 (`ko`, `en`) |

빌드 후 사용자가 앱 내 **환경설정**에서 Server URL을 변경하면, 사용자 설정이 `setting.json`보다 우선 적용됩니다.

### 3. 설치 패키지 배포

`npm run tauri:build` 실행 후 `src-tauri/target/release/bundle/` 에 생성된 설치 파일을 사용자 PC에 배포합니다.

| OS | 산출물 형식 |
| --- | --- |
| Windows | `.msi` 또는 `.exe` |
| macOS | `.dmg` |
| Linux | `.deb`, `.AppImage` 등 |

### 4. 최초 실행

<Steps>
  <Step title="앱 실행">
    설치된 Audigate Client를 실행합니다.
  </Step>
  <Step title="로그인">
    Server에 등록된 DB 사용자 계정으로 로그인합니다.
    - `OPEN`: 정상 사용
    - `LOCKED`: 승인 대기 (관리자 승인 필요)
    - `NO-USER`: 미등록 (회원가입 유도)
  </Step>
  <Step title="환경설정 확인">
    Server URL, 언어 설정이 올바른지 확인합니다.
  </Step>
</Steps>

## AudigateClient.Net (WPF) 설치

### 1. 소스 빌드

```powershell
cd D:\AG-WORKSPACE\AudigateClient.Net
dotnet restore
dotnet build
dotnet run --project src\Audigate.Client.App\Audigate.Client.App.csproj
```

### 2. Server URL 설정

`src/Audigate.Client.App/appsettings.json`:

```json
{
  "baseServerUrl": "http://192.168.0.25:9071/rest/dbuser",
  "lang": "ko"
}
```

### 3. 사용자 설정 파일

| 파일 | 용도 |
| --- | --- |
| `%AppData%\Audigate\client-settings.json` | 사용자 환경설정 (Server URL, 언어) |
| `%AppData%\Audigate\session.json` | 로그인 세션 (JWT 토큰) |

앱 내 **환경설정**에서 Server URL을 변경하면 `client-settings.json`에 저장됩니다.

### 4. 배포

```powershell
dotnet publish src\Audigate.Client.App\Audigate.Client.App.csproj -c Release -r win-x64 --self-contained
```

`bin\Release\net8.0-windows\win-x64\publish\` 폴더를 사용자 PC에 배포합니다.

## 사용 워크플로 (설치 후)

Client 설치·로그인 후 아래 순서로 **Instant Gate**를 사용합니다.

<Steps>
  <Step title="권한 요청서 생성">
    DB 서비스 → 스키마(소유자) → 오브젝트 → 권한(SELECT 등) → 기한을 지정하여 요청합니다.  
    상태: **REQUEST**
  </Step>
  <Step title="승인">
    승인자가 Server 웹 콘솔(**게이트 롤 매니저**) 또는 Client에서 결재합니다.  
    상태: **CONFIRM**
  </Step>
  <Step title="인증키 발급">
    **나의 요청 목록** → 승인된 건 선택 → **[인증 키 가져오기]** 클릭.  
    상태: **TAKEN**
  </Step>
  <Step title="Role 활성화">
    발급된 PL/SQL 스크립트를 SQL*Plus / SQL Developer에서 실행합니다.

    ```sql
    -- EE 환경
    BEGIN "AG$BROKER01".ap_role_util.apply_my_roles('<memberId>', '<authKey>'); END;
    /

    -- SE/XE 환경
    BEGIN "AG$BROKER01".ap_role_util_se.apply_my_roles('<memberId>', '<authKey>'); END;
    /
    ```
  </Step>
  <Step title="권한 확인">
    승인된 SELECT 등 허용 SQL을 실행하고, 범위 밖 SQL(INSERT 등)이 거부되는지 확인합니다.
  </Step>
</Steps>

<Note>
  `authKey`는 **요청 PC의 IP·OS·터미널**과 바인딩됩니다. 다른 PC에서 실행하면 실패할 수 있습니다. Client의 **현재 IP**와 요청 시 등록한 IP가 일치하는지 확인하세요.
</Note>

## 인증·보안

| 항목 | 내용 |
| --- | --- |
| **로그인** | SHA256 해시 비밀번호 → `POST /rest/dbuser/login` |
| **JWT** | Access Token (~1분), Refresh Token (~14일) |
| **암호화** | Blowfish 기반 대칭키 (`get-sym-key`) |
| **OS 정보** | Client가 user/host/IP/MAC 수집 → authKey 검증에 사용 |

## 설치 확인 체크리스트

| # | 확인 항목 | 방법 |
| --- | --- | --- |
| 1 | Server REST 연결 | Client 로그인 성공 |
| 2 | DB 사용자 등록 | `LOCKED`/`OPEN` 상태 (미등록 시 `NO-USER`) |
| 3 | DB 서비스 목록 | 권한 요청서에서 DB 서비스 선택 가능 |
| 4 | 권한 요청 | `REQUEST` 상태 생성 |
| 5 | 인증키 발급 | `CONFIRM` 건에서 PL/SQL 스크립트 표시 |
| 6 | Role 활성화 | `apply_my_roles` 성공, `SESSION_ROLES`에 AP-Role 1개 |

## 문제 해결

| 증상 | 원인 | 조치 |
| --- | --- | --- |
| 로그인 실패 | Server URL 오류 | `baseServerUrl`에 `/rest/dbuser` 포함 여부 확인 |
| `NO-USER` | 계정 미등록 | Server **DB 사용자 관리**에서 초대 또는 Client 회원가입 |
| `LOCKED` | 승인 대기 | 관리자에게 계정 승인 요청 |
| DB 서비스 목록 없음 | Server 미설정 | **디비 서비스 관리자**에서 Oracle 인스턴스 등록 |
| `apply_my_roles` 실패 | IP/OS 불일치 | Client **현재 IP** 확인 후 요청 IP 갱신 |
| authKey 만료 | 기한 초과 | 새 권한 요청·재승인 |
